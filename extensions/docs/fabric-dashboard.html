🧵 What is Fabric?

Fabric treats AI prompts as composable building blocks called Patterns. Instead of writing bespoke prompts every time, you pick a pattern and pipe content through it — just like Unix commands.

# Pipe anything through a pattern
echo "Your content here" | fabric --pattern extract_wisdom

# Chain patterns together
cat article.txt | fabric -p extract_wisdom | fabric -p summarize

# YouTube → AI analysis in one line
fabric -y "https://youtube.com/watch?v=..." --pattern summarize --stream
In Claude Code, you don't need the CLI at all. Just type /Fabric extract_wisdom and paste your content. The skill reads the pattern file directly and applies it natively.

📐 Pattern Anatomy

Every pattern is a markdown file with three canonical sections:

# IDENTITY and PURPOSE You are an expert at [task]. You take [input] and produce [output]. # STEPS - Step 1: Deeply analyze the input... - Step 2: Identify the key themes... - Step 3: Extract the most important... # OUTPUT INSTRUCTIONS - Only output Markdown. - Use numbered lists, not bullets. - Do not output warnings or notes. # INPUT INPUT:
Why system.md? Fabric places instructions in the AI's system message rather than user input. Testing shows this produces more reliable, consistent behavior across models.

🧠 Prompt Strategies

Strategies are meta-reasoning techniques that modify how a pattern thinks, without changing what it does. Mix any strategy with any pattern: fabric -p analyze_paper --strategy cot

CoT Chain of Thought
Step-by-step sequential reasoning. The model "thinks out loud" through each logical step before producing output. Best for complex analysis tasks.
fabric -p analyze_claims --strategy cot
ToT Tree of Thought
Explores multiple reasoning branches in parallel, evaluates each path, then converges on the best answer. Good for creative or ambiguous problems.
fabric -p summarize --strategy tot
CoD Chain of Draft
Iterative densification — produces a draft, then progressively refines it to pack more information into fewer words. Great for summaries.
fabric -p create_summary --strategy cod
AoT Atom of Thought
Decomposes the problem into atomic sub-problems, solves each independently, then synthesizes. Prevents reasoning shortcuts.
fabric -p analyze_paper --strategy aot
LtM Least to Most
Builds understanding from simple to complex. Solves the easiest sub-problem first, using each answer as context for the next. Mimics learning progression.
fabric -p explain_code --strategy ltm
REF Reflexion
Answer → Critique → Refine cycle. The model produces an answer, then critically evaluates it, then improves. Self-correcting reasoning.
fabric -p improve_writing --strategy reflexion
SC Self-Consistent
Generates multiple independent reasoning paths, then selects the answer that appears most consistently across paths. Reduces hallucination risk.
fabric -p analyze_claims --strategy self-consistent
SR Self-Refine
Initial answer → Self-critique → Optional refinement. Similar to Reflexion but lighter weight. Good for writing and formatting tasks.
fabric -p improve_writing --strategy self-refine
STD Standard
Default prompting behavior — no additional reasoning scaffolding. Direct answer. Use when the pattern itself is already well-structured.
fabric -p summarize --strategy standard
Strategies are independent of patterns. You can pair any strategy with any pattern. This separation of concerns means 239 patterns × 9 strategies = 2,151 possible combinations without writing a single new prompt.

🎬 YouTube Integration
📹 Video Processing Pipeline

Fabric has first-class YouTube support via yt-dlp. Extract transcripts, comments, and metadata, then pipe through any pattern.

1
Input URL
Any YouTube video, shorts, or playlist URL
2
yt-dlp extracts data
Transcript, timestamps, comments, metadata
3
Pattern processes content
extract_wisdom, summarize, analyze_claims, etc.
4
Structured output
stdout, clipboard, file, or pipe to next pattern
# Quick video summary fabric -y "https://youtube.com/watch?v=abc" -p summarize -s # Extract wisdom from a podcast fabric -y "URL" -p extract_wisdom --stream # Get timestamped transcript only (no AI) fabric -y "URL" --transcript-with-timestamps # Batch process a playlist fabric -y "PLAYLIST_URL" --playlist -p summarize # Get comments (requires YouTube API key) fabric -y "URL" --comments # Private/age-restricted content fabric -y "URL" --yt-dlp-args "--cookies-from-browser chrome"
Best patterns for YouTube: extract_wisdom (podcasts/talks), summarize (quick overview), create_keynote (extract presentation structure), extract_recommendations (how-to videos).
💡 Pro Tips
🔗 Power User: Chaining Patterns ▶

Unix pipes let you chain patterns. The output of one becomes the input of the next:

# Extract wisdom, then distill to micro-summary cat article.txt | fabric -p extract_wisdom | fabric -p summarize_micro # YouTube → insights → Anki flashcards fabric -y "URL" -p extract_wisdom | fabric -p create_quiz # Analyze claims, then visualize the argument structure cat debate.txt | fabric -p analyze_claims | fabric -p create_mermaid_visualization
📋 Contexts: Persistent Reference Material ▶

Contexts are saved text snippets that get injected alongside patterns. Great for company info, personal background, or project specs:

# Save a context echo "I'm a software engineer specializing in TypeScript and Swift..." > ~/.config/fabric/contexts/me.md # Use it with any pattern fabric -p improve_writing -C me # List all saved contexts fabric -x
