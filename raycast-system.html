🏗 Architecture Decisions (Feb 2026)

Why a monorepo?

Scripts and extensions are the same concern: "my Raycast customizations." One git pull syncs everything. One repo, one remote, one branch.

Why not git submodules?

Submodules require git submodule update --init --recursive on every pull. Forgetting this = the exact desync we cleaned up. Submodules solve multi-team problems we don't have.

Why not separate repos?

We had a separate loudoguno/extensions repo. It drifted — identical files, divergent histories, a nested .git on mx3, confusion about which was canonical. Two repos for one person = guaranteed drift.

🧩 New Extension

  1. Navigate to ~/code/raycast/extensions/
  2. Use Raycast "Create Extension" or copy a template:
    cp -r templates/hello-world-list my-ext
  3. Edit package.json with extension metadata
  4. npm install && npm run dev
  5. Test in Raycast (search for your command name)
  6. git add && git commit && git push
  7. On other machine: git pull && cd extensions/my-ext && npm install && npm run dev

📜 New Script Command

  1. Navigate to ~/code/raycast/scripts/
  2. Copy the template:
    cp template-script-for-raycast.sh my-script.sh
  3. chmod +x my-script.sh
  4. Edit Raycast metadata headers:
    @raycast.title, @raycast.description, etc.
  5. Script appears in Raycast immediately
  6. git add && git commit && git push
  7. On other machine: git pull (auto-reloads)

⚙️ balloons-fancy Build Process

This extension requires building a native macOS app first:

Edit Swift views
→
cd BalloonsApp && ./build.sh
→
cp -r build/BalloonsApp.app /Applications/
→
npm run dev
Must rebuild after any Swift file changes. The app must live in /Applications/ for the extension to find it.

📋 Checklist: Setting Up a New Machine

  1. Install Raycast from raycast.com
  2. git clone https://github.com/loudoguno/raycast.git ~/code/raycast
  3. Raycast > Extensions > Script Commands > Add Directories > ~/code/raycast/scripts/
  4. For each extension you want to use:
    cd ~/code/raycast/extensions/{name} && npm install && npm run dev
  5. For balloons-fancy: also build and install BalloonsApp (see above)
  6. For claude-usage: enable Safari > Develop > Allow JavaScript from Apple Events
